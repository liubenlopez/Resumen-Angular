Componente restringido por guardas

Las guardas (guards) en Angular son esencialmente funciones de middleware que se ejecutan antes de cargar una ruta específica y determinan si la ruta se puede cargar o no. Proporcionan una forma de controlar el acceso a determinadas rutas en su aplicación. Puede utilizar guardas para proteger vistas privadas o implementar guardas de ruta funcionales para manejar diversos escenarios, como comprobaciones de autenticación o autorización. Se pueden aplicar a todo el árbol de rutas hijas.

Existen varios tipos de guardas en angular, entre ella

CanActivate: Esta guarda decide si se puede activar una ruta, si devuelven verdadero, la navegación continúa.

CanActivateChild: similar a CanActivate, pero específicamente para rutas secundaria.

CanLoad: Esta guarda se utiliza para controlar la carga de un módulo.

CanDeactivate: esta protección se utiliza para controlar si un usuario puede alejarse de una ruta.

Resolver: esta protección se utiliza para obtener datos previamente antes de que se active una ruta.

Estos guardias se pueden utilizar para implementar varias funciones de seguridad y navegación en su aplicación Angular.

En este componente se aplicó CanActivate que requiere que se le asigne permiso de acceso y CanDeactivate que pide confirmación antes de salir de la ruta.
